# 一、實驗目的 (Objective)

本實驗的目的是學習如何在 **x86 Linux 主機上對 ARM 程式進行除錯 (Cross Debugging)**。
透過 **QEMU ARM Emulator** 與 **GNU Debugger (GDB)** 的結合，可以在沒有實際 ARM 硬體的情況下，模擬 ARM 環境並進行程式除錯。

本實驗主要學習：

1. 建立 Cross Debugger 環境
2. 使用 ARM cross compiler 編譯程式
3. 使用 QEMU 模擬 ARM 程式執行
4. 使用 GDB 進行 remote debugging
5. 觀察程式執行流程與 CPU registers

---

# 二、實驗環境 (Environment)

作業系統：

Ubuntu 20.04 (Virtual Machine)

使用工具：

* GNU Cross Compiler
* GNU Debugger (GDB)
* QEMU ARM Emulator

主要工具：

| 工具                      | 用途          |
| ----------------------- | ----------- |
| arm-linux-gnueabihf-gcc | 編譯 ARM 程式   |
| gdb-multiarch           | 跨平台除錯工具     |
| qemu-arm                | ARM CPU 模擬器 |

# 三、實驗程式 (Test Program)

測試程式 `test.c`：

```c
#include <math.h>
#include <stdio.h>

int main(void)
{
    int a;
    double b;

    a = 10;
    b = 20 + cos((double) a);

    printf("%f\n", b);

    return 0;
}
```

此程式會計算：

```
b = 20 + cos(a)
```

並輸出計算結果。

---

# 四、程式編譯 (Compilation)

使用 ARM Cross Compiler 編譯程式：

```bash
arm-linux-gnueabihf-gcc -static -g -lm test.c -o test.exe
```

參數說明：

| 參數      | 說明                   |
| ------- | -------------------- |
| -static | 靜態連結                 |
| -g      | 加入 debug information |
| -lm     | 使用 math library      |
| -o      | 指定輸出檔案               |

編譯完成後會產生：

```
test.exe
```

此檔案為 **ARM executable binary**。

# 五、Remote Debugging 流程

本實驗使用 **QEMU + GDB remote debugging**。

架構如下：

```
x86 Linux PC
│
├── GDB (Debugger)
│
└── QEMU (ARM Emulator)
        │
        └── ARM Program (test.exe)
```

---

## Step 1：啟動 QEMU

在 Terminal 1 執行：

```bash
qemu-arm -g 12345 ./test.exe
```

參數說明：

| 參數       | 說明                      |
| -------- | ----------------------- |
| -g 12345 | 啟動 gdbserver 並等待 GDB 連線 |

此時程式會 **暫停並等待 GDB 連線**。

---

## Step 2：啟動 GDB

在 Terminal 2 執行：

```bash
gdb-multiarch test.exe
```

進入 GDB 介面：

```
(gdb)
```

---

## Step 3：連線至 QEMU

```bash
target remote localhost:12345
```

GDB 會連接到 QEMU ARM emulator。

---

## Step 4：設定 Breakpoint

```bash
break main
```

設定程式在 `main()` 停止。

---

## Step 5：開始執行

```bash
continue
```

程式會執行到 `main()` 停止。

# 六、GDB Debug 指令練習

## 1. 單步執行

```bash
n
```

功能：

```
next
執行下一行程式
```

---

## 2. 進入函式

```bash
s
```

功能：

```
step
進入函式內部
```

---

## 3. 查看變數

```bash
p a
p b
```

輸出程式變數值。

---

## 4. 查看 CPU Registers

```bash
info register
```

可以觀察 ARM CPU registers，例如：

```
r0
r1
r2
sp
pc
lr
```

其中：

| Register | 說明                |
| -------- | ----------------- |
| r0-r12   | General registers |
| sp       | Stack Pointer     |
| pc       | Program Counter   |
| lr       | Link Register     |

---

## 5. 查看記憶體

```bash
x/10x $sp
```

功能：

查看 stack memory。

# 八、問題回答 (Questions)

## 1. What is JTAG ?

JTAG (Joint Test Action Group) 是一種硬體除錯與測試介面標準，用於嵌入式系統與 IC 測試。

JTAG 主要功能：

* Debug CPU
* 檢查硬體連線
* 燒錄 firmware
* 觀察 CPU registers
* 設定 breakpoints

在嵌入式系統開發中，JTAG 常用於直接對 MCU 或 SoC 進行硬體層級除錯。

---

## 2. What is GDB Server ?

GDB Server 是一個 **在 target device 上運行的程式**，負責：

* 接收 GDB 指令
* 控制程式執行
* 傳回 debug 資訊

GDB Server 會與 GDB Host 使用 **Remote Serial Protocol (RSP)** 通訊。

架構：

```
GDB (host)
    │
    │ remote protocol
    │
GDB server (target device)
    │
    └── target program
```

---

## 3. What is GDB Stub ?

GDB Stub 是一個嵌入在程式中的 debug agent。

其功能：

* 接收 GDB 指令
* 控制程式執行
* 與 GDB Host 通訊

GDB Stub 通常會直接被編譯進程式中。

---

## 4. Difference between GDB Server and GDB Stub

| 項目   | GDB Server              | GDB Stub           |
| ---- | ----------------------- | ------------------ |
| 位置   | 獨立程式                    | 嵌入在程式              |
| 執行方式 | 外部執行                    | 程式內部               |
| 控制方式 | 控制 target program       | 程式自己控制             |
| 使用情境 | Linux / Embedded system | Bare-metal systems |

簡單來說：

* **GDB Server：外部 debug agent**
* **GDB Stub：內嵌 debug agent**
